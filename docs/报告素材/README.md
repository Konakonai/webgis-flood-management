# 报告写作素材索引

本目录只记录项目采用的方法、实现结果和可验证证据，用于后续自行撰写个人报告与小组总体设计报告。内容不讨论系统早期状态，也不按正式报告口吻扩写。

## 文件说明

- `同学A-前端GIS与Podman部署.md`：冯峥嵘可用于个人报告的工作素材，覆盖任务分配中的同学A、全部前端整合与Podman部署。
- `同学B-后端业务与数据服务.md`：按任务分配归入同学B的后端工作素材，姓名和学号留给组长填写。
- `小组总体设计素材.md`：适合写入总体设计报告的系统目标、总体架构、功能组成、数据设计、接口协作与验证结果。
- `插图缺口与制作方案.md`：对照“杭州智能旅游服务系统总体设计报告”中的图片类型，列出需要补拍的界面图和需要用Python绘制的说明图。

## 写作建议

每项工作可按“目标—方法—效果—证据”展开：先说明要解决的业务问题，再写采用的技术方法，随后写形成的用户效果或系统能力，最后引用代码、接口、测试或运行截图作为证据。避免只罗列框架名称，也不要把开发过程写成问题清单。
