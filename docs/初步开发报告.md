## 摘要

项目已形成可直接构建、运行和演示的完整交付包。系统采用 Vue 3、Spring Boot、PostgreSQL/PostGIS、OSRM 与 Nginx，提供地图空间检索、监测与预警服务、应急资源和工单调度、公众移动上报、后台维护 API、操作审计与实时消息。

本轮完成了前端真实数据接入、HCI 收尾和完整容器验收。前端请求 Mock 已从仓库移除；调度台通过 JWT 访问数据库工单，使用 OSRM 江苏道路数据规划路线，并通过后端事务完成资源占用、抵达和释放。窄屏通过工作区切换避免双面板重叠，抢险资源列表具备独立滚动区域。四个 Compose 服务已从空卷冷启动并全部达到健康状态。

## 二、技术架构

| 层次 | 组件 | 职责 |
|---|---|---|
| 前端 | Vue 3、TypeScript、MapLibre GL JS、Turf.js、Pinia、Naive UI | 地图、绘制缓冲、查询联动、调度和移动上报 |
| 接入层 | Nginx | 静态页面、同源代理、限流、安全响应头 |
| 业务层 | Spring Boot 3.5.16、Spring Security、MyBatis-Plus、STOMP | REST、JWT/RBAC、状态机、审计、定时模拟和广播 |
| 数据层 | PostgreSQL 15、PostGIS 3.4、Flyway | 关系数据、空间几何、索引、约束和版本迁移 |
| 路由层 | OSRM 5.27.1 MLD | 基于江苏 OpenStreetMap 数据提供道路路径 |
| 部署层 | Podman/Docker Compose、命名卷 | 服务编排、健康检查、持久化与重启恢复 |

浏览器统一访问 `http://127.0.0.1:8088`。Nginx 将 `/api`、`/ws`、`/uploads` 和 `/osrm` 分别转发到相应服务，前端不需要跨域地址。

## 三、功能完成范围

### 3.1 地图与空间查询

- OSM/Carto 底图、主题切换、业务图层和属性弹窗；
- 矩形、多边形绘制与 Turf 缓冲；
- 名称、设施类型和空间范围组合查询；
- 后端使用参数化 PostGIS `ST_Intersects`，返回 GeoJSON；
- 查询列表、地图 Marker 与定位联动；
- 绘图和右键登记采用互斥交互模式。

### 3.2 监测与风险预警

- 监测站点、最新数据、历史过程和汇总接口；
- 定时生成水位、雨量、流量模拟数据；
- 配置驱动阈值与自动预警；
- 预警确认、驳回、发布、撤销、统计和时间线；
- STOMP 推送监测、预警和工单变化。

监测、预警、配置和日志已经具备完整 API/OpenAPI 管理能力。当前地图前端重点呈现空间检索与应急调度；独立态势大屏和运营后台可作为展示层增强，不影响现有交付链路运行。

### 3.3 应急调度

- JWT 登录和 ADMIN/OPERATOR/VIEWER 角色控制；
- 真实资源、泵车和工单加载；
- 地图右键创建服务器工单；
- 只从空闲泵车中计算最近候选；
- OSRM 返回真实道路 GeoJSON、距离和时间；
- 工单与资源在数据库事务内加锁派发；
- 抵达现场单独记录 `arrivedAt`；
- 完成或拒绝后释放应急资源；
- `/topic/work-orders` 通知其他客户端刷新。

### 3.4 公众上报

- HTML5 定位与地图中心取点；
- 积水深度和描述；
- PNG/JPEG 选择、长边 1920px 压缩与 5MB 客户端边界；
- 服务端实际格式、像素、大小、数量再次校验；
- 服务端生成 `FR-...` 追踪码；
- 追踪状态、历史、结果和图片；
- 浏览器只保存追踪码，不保存图片或伪造状态。

### 3.5 后台与数据维护

- 用户资料、角色、启停与最后管理员保护；
- 监测站点和应急资源 CRUD；
- GeoJSON 图层上传、拓扑校验与 PostGIS 入库；
- 阈值配置和审计日志；
- OpenAPI/Swagger 作为当前维护入口。

## 四、数据与并发设计

数据库由 V1—V4 迁移维护，主要表包括用户角色、监测站点、监测数据、预警、应急资源、工单、状态历史、附件、图层与空间设施。V4 将初始工单规范为真实业务语义的模拟用户记录，并删除浏览器验收残留数据。

关键约束：

- WGS84 / EPSG:4326 全链路一致；
- 业务几何建立 GiST 空间索引；
- 类型、状态、坐标、数量、水深由 CHECK 约束兜底；
- 处理中工单对资源 ID 建立部分唯一索引；
- 派单、状态迁移、图片追加和管理员启停使用数据库行锁；
- 事务提交后才发送 WebSocket 消息；
- 附件在数据库回滚或删除工单时同步清理。

## 五、安全边界

- BCrypt 密码与 24 小时 JWT；
- 每次请求重新读取用户启用状态和角色，旧 Token 会立即受账号变更影响；
- 工单和联系方式仅 ADMIN/OPERATOR 可见；
- STOMP CONNECT 必须携带 Bearer Token，客户端 SEND 被拒绝；
- GeoJSON、查询名称、分页、枚举、坐标和文件均有限额；
- Popup 动态字段统一 HTML 转义；
- Nginx 对登录、空间查询、匿名上报和图片追加限流；
- Nginx 设置 CSP、nosniff、frame 与 referrer 安全头；
- 后端和 OSRM 使用非 root 用户与 `no-new-privileges`；
- 默认所有宿主端口只绑定 `127.0.0.1`。

初始管理员账号和 `.env.example` 的密码只适用于本机首次部署。任何共享部署都必须替换数据库密码、JWT 密钥和默认账号，并增加 HTTPS。
